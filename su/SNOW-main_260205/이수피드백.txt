
1. display:flex가 불필요하게 많이 사용되었어요. 
/////////////////////
display: flex;
flex-direction: column;
align-items: center;
////////////////////
이런 설정이 많은데 각 영역과 요소는 기본적으로 세로 나열이기 때문에 저 설정이 필요하지 않아요. 
flex는 영역 내의 여러 개의 요소를 배치할 때 사용하는데, 영역안에 요소가 없는데도 flex가 설정되어 있어요. flex를 왜 사용해야하는지 구분하는 게 필요할 것 같습니다. 

2. <button>태그를 꼭 p태그 안에 넣지 않아도 되요. 
또한 버튼은 기능에 따라 a태그를 버튼처럼 디자인해서 사용하는 걸 추천해요. 
button태그는 버튼의 기능을 스크립트로 만들어서 연결하는 것까지 고려해야 해요 

3. CSS에 px로 고정된 값이 너무 많고 position이 많이 사용되었어요. 
화면 크기와 상관없이 무조건 px의 고정값이 들어가니 레이아웃이 자연스러울 수 없어요. 
특히, 여백이 -1000px 이상이거나 몇 백px의 고정 여백이 들어가는 건 꼭 사용이 필요한지 확인해봐야 해요. section에 margin-bottom: -1035px; 이런 설정은 좋은 방법은 아니예요. 
또한 각 영역이 position으로 설정되다 보니 크기나 공간 설정에도 문제가 되니까, position을 사용해야 하면 레이아웃 설계부터 고민해 보는 게 좋습니다. 

4. reset.css에서 전체 선택자에 box-sizing이 기본값으로 설정되어 있어요. 
전체요소 선택자는 기본값 css 설정을 최소화해요. 기본값 설정이 불필요한 요소까지 모두 공통의 css가 적용되는 것이라 나중에는 css 로딩 속도에도 영향을 줍니다. 실제로는 전체선택자를 사용하지 않기도 하고요! 

5. 전반적으로 큰 모니터에서도 작은 모니터에서도 모든 내용이 크게 들어가 있어서 디자인 시안의 분위기가 나오려면 구성요소의 크기 조절이 필요해요. 
제가 코드 수정하면서 필요한 부분은 최소로 크기 수정을 하긴 했는데 주변 요소와 크기 균형이 맞을 수 있게 조절해 보세요!
